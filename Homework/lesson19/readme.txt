1. Створити файл XML і відповідну йому схему XSD. (виберіть один з наведених варіантів)
2. При розробці XSD використовуйте прості та складні типи, перечисления, шаблони і граничні значення.
3. Згенерувати клас, відповідний даному опису.
4. Створити додаток(клас) для розбору XML-документів та ініціалізації коллекції об'єктів інформацією з XML-файлу.
Для розбору використовуйте парсери SAX, DOM і StAX. Для сортування об'єктів використовуйте інтерфейс Comparator.
5. Провести перевірку XML-документа із залученням XSD.
6. Перетворити в JSON

варіанти:
А  Банківські внески.
- Name - назва банку;
- Country - країна реєстрації;
- Type - тип вкладу (терміновий, розрахунковий, накопичувальний, ощадний, металевий);
- Depositor - ім'я вкладника;
- Account id - номер рахунку;
- Amount on deposit - сума вкладу;
- Profitability - річний відсоток;
- Time constraints - термін вкладу.

Кореневий елемент називається Bank.



Б  Пиво.
- Name - назва;
- Type - тип пива (темне, світле, табір, живе);
- Al - алкогольне або безалкогольне;
- Manufacturer – фірма-виробник;
- Ingredients (має бути кілька) - інгредієнти: вода, солод, хміль,
цукор тощо;
- Chars (має бути кілька) - характеристики: кількість оборотів
(якщо алкогольне), прозорість (у відсотках), фільтроване чи ні,
харчова цінність (ккал), спосіб розливу (обсяг та матеріал ємностей).

Кореневий елемент називається Beer.

В Літаки.
Літаки можна описати за такою схемою:
- Model - назва моделі;
- Origin - Країна виробництва;
— Chars (має бути кілька) — характеристики, які можуть бути такими: тип (пасажирський, вантажний, поштовий, пожежний, сільськогосподарський), кількість місць для екіпажу, характеристики (вантажопідйомність, кількість пасажирів), наявність радара;
- Parameters - довжина (в метрах), ширина (в метрах), висота (в метрах);
- Price - ціна (у доларах, гривнях).
